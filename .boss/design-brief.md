# 设计简报: 360安全龙虾驾驶舱（OpenClaw Agent Cockpit）

## 一句话描述
一个面向运维人员和AI工程师的Agent监控和管理平台,用"人话"实时监控、调度、理解 AI Agent 的运行状态、成本、产出与风险。

## 目标用户
- 主要用户: AI工程师、运维人员、技术管理者
- 用户特点: 需要监控多个AI Agent的运行状态,关注成本、效率和异常,希望用可视化的方式快速了解系统状态

## 核心场景
1. 用户打开驾驶舱 → 在总览页看到所有Agent的实时状态、今日任务完成情况、异常告警 → 快速判断系统是否正常运行
2. 用户点击某个Agent卡片 → 查看详细的执行历史、token消耗、响应时间趋势 → 定位性能瓶颈或异常
3. 用户在任务页查看当前排程 → 管理定时任务和自动化流程 → 调整执行策略

## 功能范围
### 第一版必须有
- **总览页（核心）**
  - Top指标卡片（活跃Agent、完成任务数、成功率、响应时间、token消耗、成本、告警数）
  - 趋势图（token消耗、任务量、响应时间）
  - 环形图（任务状态分布、队列分布、模型占比）
  - Agent卡片（拟人化展示每个Agent的状态）
  - 任务管理（当前任务、最近任务）

- **资源用量页**
  - 请求数统计
  - token使用分析
  - AI用量展示
  - Cron占比
  - 任务占比

- **Agent详情页**
  - 基本信息（头像、名称、职责、状态）
  - 核心指标卡片
  - 当前任务
  - 异常信息
  - 执行历史表格
  - token趋势图
  - 产出记录

- **记忆页**
  - 分类导航
  - 文件列表
  - 内容编辑器

- **文档页**
  - 文档概览
  - 文档工作台
  - Markdown编辑区

- **任务页**
  - 排程卡片
  - Cron看板
  - 任务列表

- **设置页**
  - 基础配置

- **全局功能**
  - 深色/浅色模式切换
  - 中英文切换
  - 全局搜索（Agent、任务、文档）
  - 右侧控制栏（告警中心、任务队列、定时任务）

### 明确不做（留给后续版本）
- Agent权限管理
- 多租户支持
- 数据导出功能
- 邮件/钉钉告警推送
- 自定义大屏配置
- Agent性能优化建议（AI分析）

## 成功标准
- 用户能在3秒内判断系统是否正常
- 所有图表加载有平滑动画
- 深色模式切换体验流畅
- Agent卡片有拟人化的状态表达
- 响应式布局适配各种屏幕尺寸
- 所有数据用"人话"展示（避免技术黑话）

## 技术栈要求
- Next.js 14 (App Router)
- TypeScript
- TailwindCSS
- shadcn/ui
- ECharts（图表）
- Framer Motion（动效）
- Zustand（状态管理）

## 设计风格要求
- 未来感 + 企业级 + 高级灰 + 轻动效 + 数据可视化导向
- 对标：飞书数据看板（信息密度）、Linear（干净高级）、Vercel（现代感）、Datadog（监控感）
- 主色：蓝色系（#3B82F6, #6366F1）
- 强调数据可读性和层级
- 支持深色模式（必须重新设计，不是简单反色）
- 所有图表必须有动画
- Agent卡片必须拟人化设计（像员工）
